# 本机翻译版使用说明

本版本采用“网页工具 + 本机后台翻译服务”的方式运行。招股书段落只会发送到用户自己电脑上的 `127.0.0.1` 翻译接口，不会发送到云端服务器。

## 第一次安装

### macOS

1. 双击 `install_local_translator.command`。
2. 等待脚本安装 Python 依赖和英译中模型。
3. 看到 `Installation is complete.` 后关闭窗口。

如果 macOS 阻止运行，可以右键点击脚本，选择“打开”。

macOS 的本机翻译运行环境会安装到：

```text
~/Library/Application Support/Prospectus Local Translator
```

这样 App 不需要读取桌面文件夹里的隐藏 Python 环境，能减少 macOS 权限限制导致的启动失败。

安装脚本还会注册一个 macOS 后台服务：

```text
~/Library/LaunchAgents/com.prospectus.localtranslator.plist
```

安装完成后，本机翻译服务会在后台启动，并在以后登录电脑时自动启动。

### Windows

1. 双击 `install_local_translator.bat`。
2. 等待脚本安装 Python 依赖和英译中模型。
3. 看到 `Installation is complete.` 后关闭窗口。

安装脚本会创建一个 Windows 登录自启任务 `Prospectus Local Translator`，让本机翻译服务在后台运行。

## 日常使用

安装完成后，直接在浏览器打开网站：

```text
https://kunlins.github.io/verification-margin-note-tool/
```

网页打开后，点击“检查本机翻译”。如果显示“本机翻译：可用”，即可直接使用自动翻译。

`Prospectus Local Translator.app`、`start_tool.command` 和 `start_tool.bat` 仍保留为备用入口，但日常使用不再需要它们。

## 网站开关

如果网页部署在 GitHub Pages，请把 `status.json` 和网页一起上传到同一个网站目录。

App 每次打开时会先检查：

```text
网站地址/status.json
```

只有 `enabled` 为 `true` 时，App 才会启动本机翻译服务并打开网站。

正常开放时：

```json
{
  "enabled": true,
  "message": "Service available."
}
```

需要关闭服务时，把网站上的 `status.json` 改成：

```json
{
  "enabled": false,
  "message": "This service is no longer available."
}
```

普通用户再次打开 App 时会看到该提示，并且 App 不会启动本机翻译服务。

## 翻译工作流

1. 上传 PDF。
2. 单击或拖拽圈选招股书段落。
3. 如果勾选了“新 note 自动翻译”，系统会把英文段落发到本机翻译服务，并把中文结果填入右侧“中文翻译”框。
4. 用户可以人工修改中文翻译。
5. 点击“导出 Word 验证清单”时，如果勾选了“导出前补齐翻译”，系统会先批量翻译仍为空的中文翻译项，再导出 Word。

## 中国内地网络说明

日常使用不需要 VPN，因为翻译在本机运行。

第一次安装时需要下载 Python 依赖和 Argos 英译中模型。如果下载失败，通常是因为访问 PyPI、GitHub 或模型源不稳定。可以在网络条件较好的环境安装一次，或改用内网镜像/预置模型包。

如果安装时出现 `CERTIFICATE_VERIFY_FAILED`，通常是 Python 没有找到可用的 HTTPS 根证书。当前安装脚本会优先使用 `certifi` 证书包；请重新运行一次安装脚本。如果仍失败，可以在 macOS 的 Python 安装目录里运行 `Install Certificates.command` 后再试。

## 注意事项

- Argos / LibreTranslate 适合作为中文翻译初稿，法律和财务披露语境仍建议人工复核。
- 当前 HTML 仍通过 CDN 加载 PDF.js、pdf-lib 和 JSZip。如果需要完全离线版本，还需要把这些前端库下载到本地并改成相对路径引用。
- 本机服务只监听 `127.0.0.1`，同一局域网内其他电脑不能访问。
- 当前 App 是未签名的本地 App，不需要 Apple Developer license。如果通过互联网分发给其他 Mac，第一次打开时可能需要右键点击 App 并选择“打开”。
